# AutoCodeHarness 入门指南

欢迎使用 **AutoCodeHarness** - 你的 Rust 自主研究框架!

## 刚刚创建了什么

一个完整的 Rust 项目,包含:

- ✅ **核心库** - 类型系统和模块结构
- ✅ **研究引擎** - 假设生成和实验管理  
- ✅ **执行框架** - 资源限制和监控
- ✅ **代码库智能** - AST 分析和依赖关系
- ✅ **集成层** - Git、MCP 服务器、Codex CLI
- ✅ **完整文档** - 专为 AI 智能体优化(AGENTS.md)
- ✅ **研究指南** - 策略和最佳实践
- ✅ **技能模板** - 可复用工作流
- ✅ **Git 仓库** - 已完成初始提交

总计: **约 3,229 行代码**,18 个文件

## 快速开始(3 步)

### 1. 构建项目

```bash
cd /tmp/autocodeharness
cargo build --release
```

这将编译 Rust 代码并下载依赖(约 197 个包)。

### 2. 运行测试

```bash
cargo test
```

应该看到类似输出:
```
running 2 tests
test tests::test_metrics_comparison ... ok
test tests::test_simplicity_tiebreaker ... ok
```

### 3. (可选) 测试 MCP 服务器

```bash
cargo run --bin mcp-server
```

## 与 Claude Code 集成

### 添加为 MCP 服务器

```bash
# 从任何目录,添加 AutoCodeHarness MCP 服务器
claude mcp add autocodeharness -s user -- cargo run --bin mcp-server --manifest-path /tmp/autocodeharness/Cargo.toml

# 重启 Claude Code 以激活
```

### 在聊天中使用

集成后,Claude Code 可以:
- 基于代码库分析生成研究假设
- 自主设计和运行实验
- 跟踪结果并做出保留/丢弃决策
- 在你睡觉时通宵迭代

## 与 Codex 集成(可选)

用于多模型协作(Claude Code 执行 + GPT-5.4 审查):

```bash
# 设置 Codex(如果还没有)
codex setup  # 提示时选择 gpt-5.4

# 将 Codex MCP 服务器添加到 Claude Code
claude mcp add codex -s user -- codex mcp-server

# 现在可以让 Claude Code 获取 GPT-5.4 审查
```

## 架构亮点

### 1. 类型优先设计

所有核心类型定义在 `src/lib.rs`:
- `Hypothesis` - 带理由的研究假设
- `Experiment` - 完整实验配置
- `ExperimentResult` - 带指标的结果
- `Metrics` - 性能测量
- `CodeChange` - 代码修改规范

### 2. 全程 Async/Await

基于 Tokio 运行时实现高效 I/O:
- 并发实验执行
- 非阻塞 Git 操作
- 异步 MCP 服务器

### 3. 强错误处理

到处使用 `anyhow::Result<T>`:
- 用 `?` 显式错误传播
- 生产代码中无 panic
- 有意义的错误消息

### 4. 尽可能零拷贝

利用 Rust 所有权:
- 引用而不是克隆
- 共享数据用智能指针
- 热路径最少分配

### 构建错误

```bash

cargo clean
cargo build
```

### Git 问题

```bash
# 检查 git 状态
cd /tmp/autocodeharness
git status
git log --oneline
```

### MCP 服务器连接失败

```bash
# 检查服务器是否启动
cargo run --bin mcp-server

# 检查 Claude Code MCP 配置
claude mcp list

# 删除并重新添加
claude mcp remove autocodeharness
claude mcp add autocodeharness -s user -- cargo run --bin mcp-server --manifest-path /tmp/autocodeharness/Cargo.toml
```
